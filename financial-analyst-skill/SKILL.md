# 財務分析師（Financial Analyst）

把一次投資分析或配置健檢完整跑完，並**真正寫成使用者電腦上的 Markdown 檔案**——這是
artifact 儀表板做不到的部分（artifact 活在瀏覽器沙盒，碰不到檔案系統；這個 skill 在
Cowork 環境裡有檔案系統／裝置橋接，所以由它負責落地）。

這個 skill 有兩種模式，先判斷使用者要哪一種：

- **模式 A · 標的分析**：目標是一支股票／ETF／公司／產業／主題。
- **模式 B · 配置健檢**：目標是「一整組資產配置」的健康度（例如「60% 0050、30% 台積電、
  10% 現金」），評估風險高低並給調整方向，且會跟**上一次記錄**做比較。

## 何時使用

使用者說類似這些話時觸發：
- 「分析 00922 並存檔」「幫我研究台積電」
- 「分析 2330，用芒格和塔勒布的視角」
- 「把這支股票的分析存成 md 檔」
- 「看一下光通訊產業，存起來」
- 「幫我健檢一下配置：60% 0050、30% 台積電、10% 現金」
- 「配置健檢並存本地」「這次配置跟上次比怎麼樣」

若使用者的話同時包含一個標的和「配置」「健檢」字樣，優先判斷：如果內容是「一組資產的
組合／權重描述」→ 模式 B；如果是「單一標的」→ 模式 A。不確定就直接問使用者。

## 存檔位置（重要，兩種模式共用）

這個 skill 跑在 Cowork 的雲端容器裡，容器本身是暫時的，**必須透過裝置橋接把檔案寫回
使用者電腦**才算真正「本地存檔」：

1. 呼叫 `device_list_dir` 確認目前連接的資料夾裡，是否有名為「蒸餾你的投資顧問」的專案
   資料夾（範例路徑：`/Users/sophia.lee/蒸餾你的投資顧問`）。若有，這就是存檔的根目錄。
2. 若該資料夾底下已有 `analyses/` 子資料夾，用 `device_stage_files` 把要更新的那個檔案
   （模式 A：`analyses/{目標}.md`；模式 B：`analyses/配置健檢.md`）抓進容器讀取舊內容，
   準備往下追加；若檔案還不存在，視為第一次寫入。
3. 在容器內（例如 `/tmp/`）組好完整的新檔案內容（舊內容 + 追加的新段落）。
4. 呼叫 `SendUserFile` 把組好的檔案送出，拿到 `file_uuid`。
5. 呼叫 `device_commit_files`，把 `file_uuid` 寫回 `/Users/sophia.lee/蒸餾你的投資顧問/analyses/{檔名}`
   （若步驟 1 找到的路徑不同，用實際路徑）。
6. 若完全沒有連接到任何裝置（`device_list_dir` 失敗），改成只在 Cowork 容器工作目錄寫檔、
   用 `SendUserFile` 送出，並明確告訴使用者：「目前沒有連到你的電腦，這個檔案只送到對話
   視窗，沒有存進『蒸餾你的投資顧問』資料夾，等連上裝置後我可以幫你補存。」不要假裝已經
   存進本機。

## 鏡頭分兩類

**分析角度（資料型，內建）：**
- `個股分析`（僅模式 A）— 值不值得關注、估值貴不貴、關鍵驅動因子
- `財報季`（僅模式 A）— 財報前/後、市場預期、關鍵指標、可能的意外
- `產業觀察`（僅模式 A）— 競爭格局、龍頭是誰、趨勢轉折點
- `配置健檢`（僅模式 B，預設啟用，不用使用者指定）— 用煞車（防守型資產）跟油門
  （進攻型資產）的邏輯評估配置風險高低，結尾給「調整方向」（是選項，不是指令）

**顧問團（呼叫已安裝的 nuwa skill，正版蒸餾，兩種模式都可疊加）：**
- `芒格` → 呼叫 **munger-perspective**（逆向思考、多元思維模型、能力圈）
- `塔勒布` → 呼叫 **taleb-perspective**（反脆弱、尾部風險、非對稱下注）
- `Naval` → 呼叫 **naval-perspective**（特定知識、槓桿、長期複利）

> 重要：顧問視角**一定要透過已安裝的 nuwa skill** 產生，不要自己憑印象模擬。若對應的
> skill 沒安裝，明確告訴使用者「該顧問 skill 尚未安裝，請先安裝或改用其他鏡頭」，不要
> 用內建印象硬湊。

## 執行步驟

### 1. 解析請求
從使用者的話裡抓出：
- **模式**：A（標的分析）或 B（配置健檢）
- **目標**（必要）：模式 A 例如「00922」「台積電」「光通訊」；模式 B 例如
  「60% 0050、30% 台積電、10% 現金」
- **鏡頭**（可選）：若使用者有指定就用指定的；模式 A 沒指定時預設用「個股分析」；
  模式 B 一定包含內建的「配置健檢」鏡頭，顧問鏡頭則只在使用者要求時加上。
  使用者可同時要多個（例如「個股分析＋芒格＋塔勒布」）。

若目標不明確，先問清楚再繼續。

### 2. 依序產生每個鏡頭的分析
逐一處理選定的鏡頭（一次一個，避免同時大量請求）：

- **資料型鏡頭**：先用網路搜尋抓最新行情/財報/新聞，再分析。每段控制在約 200 字，
  結論先行，附一個主要風險，模式 A 再加一個反方觀點、模式 B 再加一個調整方向，用
  繁體中文、乾淨純文字。
- **顧問型鏡頭**：呼叫對應的已安裝 nuwa skill（例如「用芒格的視角分析 00922」，或模式 B
  用「用芒格的視角看看這個配置：60% 0050、30% 台積電、10% 現金」），讓那個 skill 用
  蒸餾框架回答。把它的輸出原樣收進來。

### 3. 寫一句「本次重點」
綜合這次所有鏡頭，用**一句話**（40 字內）總結這次分析／健檢的核心結論。放在這次紀錄
的最上面。

### 4.（僅模式 B）與上次比對
讀取 `analyses/配置健檢.md` 目前的最後一筆紀錄（若存在）：
- 比較這次的配置描述跟上次的配置描述有沒有變化（權重、標的組成）。
- 比較這次「配置健檢」鏡頭的結論／風險高低跟上次是否一致。
- 用一句話寫成「與上次比對：」（例如「與上次比對：現金部位從 20% 降到 10%，防守力道
  略減，風險評分持平」）。若是第一次寫入（沒有上次紀錄），這行寫「與上次比對：這是第
  一次記錄，尚無比較基準」。

### 5. 追加寫入 Markdown 檔
- **模式 A** 檔名：`analyses/{目標}.md`（例如 `analyses/00922.md`）。把目標裡不適合當
  檔名的字元換成底線。
- **模式 B** 檔名固定：`analyses/配置健檢.md`（單一檔案，累積所有次數的健檢紀錄，方便
  照時間序比較同一組配置怎麼演變）。
- 若 `analyses/` 資料夾不存在，先建立。
- 若該檔**不存在**，先寫檔頭（見範本），再寫本次紀錄。
- 若該檔**已存在**，把本次紀錄**追加到檔尾**（不要覆蓋舊紀錄）——這樣歷史會一路累積，
  方便日後對照「上次怎麼看、這次怎麼看」。
- 依照上面「存檔位置」章節的步驟，把更新後的完整檔案內容送回使用者電腦。

### 6. 回報
告訴使用者：存到哪個檔（含資料夾路徑）、這是第幾次分析／健檢、本次重點那一句、
（模式 B）與上次比對那一句。

## Markdown 檔範本

新檔第一次寫入時，先放檔頭：

**模式 A：**
```markdown
# {目標} 分析紀錄

> 由「財務分析師」skill 自動產生。僅供分析參考，不構成投資建議。
> 顧問視角為 nuwa 蒸餾框架，非本人真實想法。

---
```

**模式 B：**
```markdown
# 配置健檢紀錄

> 由「財務分析師」skill 自動產生。僅供分析參考，不構成投資建議。
> 顧問視角為 nuwa 蒸餾框架，非本人真實想法。

---
```

之後每次分析追加一段（注意最前面留一個空行分隔）：

**模式 A：**
```markdown

## {YYYY-MM-DD HH:mm}

**本次重點：** {一句話總結}

**使用鏡頭：** {列出這次用的鏡頭，例如 個股分析、芒格}

### 個股分析
{內文}

### 芒格
{munger-perspective 的輸出}

...（其餘鏡頭依序）

---
```

**模式 B：**
```markdown

## {YYYY-MM-DD HH:mm}

**本次重點：** {一句話總結}

**目前配置：** {使用者描述的配置}

**與上次比對：** {一句話比較}

**使用鏡頭：** {列出這次用的鏡頭，例如 配置健檢、芒格}

### 配置健檢
{內文，結尾是「調整方向：」}

### 芒格
{munger-perspective 的輸出}

...（其餘鏡頭依序）

---
```

## 邊界與注意

- **只做分析與寫檔，不執行任何交易操作。**
- 顧問視角必須來自已安裝的 nuwa skill；沒裝就說沒裝，不要假裝。
- 行情/財報數字一定要先查最新再寫，並在內文註明資料日期。
- 資產配置一律用相對權重表達，不詢問使用者實際金額。
- 存檔一定要走「存檔位置」章節的裝置橋接流程，確認真的寫進使用者電腦，不要只在容器
  裡寫完就回報「已存檔」。
- 這個 skill 負責「真正寫檔」；視覺化互動仍可用「蒸餾你的分析師」artifact 儀表板，兩者
  共用同一套分析邏輯，但儲存位置不同（artifact 存瀏覽器沙盒、此 skill 存本機資料夾）。
