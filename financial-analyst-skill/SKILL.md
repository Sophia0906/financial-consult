# 財務分析師（Financial Analyst）

把一次投資分析完整跑完，並**真正寫成本機 Markdown 檔案**——這是 artifact 儀表板做不到的部分（artifact 活在瀏覽器沙盒，碰不到檔案系統；這個 skill 在 Cowork 環境裡有檔案系統，所以由它負責落地）。

## 何時使用

使用者說類似這些話時觸發：
- 「分析 00922 並存檔」「幫我研究台積電」
- 「分析 2330，用芒格和塔勒布的視角」
- 「把這支股票的分析存成 md 檔」
- 「看一下光通訊產業，存起來」

## 這個 skill 做什麼

給定一個**目標**（股票代號、公司、ETF、產業或主題）和一組**鏡頭**，依序產生分析，然後**追加寫入**該目標對應的 Markdown 檔。

### 鏡頭分兩類

**分析角度（資料型，內建）：**
- `個股分析` — 值不值得關注、估值貴不貴、關鍵驅動因子
- `財報季` — 財報前/後、市場預期、關鍵指標、可能的意外
- `產業觀察` — 競爭格局、龍頭是誰、趨勢轉折點

**顧問團（呼叫已安裝的 nuwa skill，正版蒸餾）：**
- `芒格` → 呼叫 **munger-skill**（逆向思考、多元思維模型、能力圈）
- `塔勒布` → 呼叫 **taleb-skill**（反脆弱、尾部風險、非對稱下注）
- `Naval` → 呼叫 **naval-skill**（特定知識、槓桿、長期複利）

> 重要：顧問視角**一定要透過已安裝的 nuwa skill** 產生，不要自己憑印象模擬。若對應的 skill 沒安裝，明確告訴使用者「munger-skill 尚未安裝，請先安裝或改用其他鏡頭」，不要用內建印象硬湊。

## 執行步驟

### 1. 解析請求
從使用者的話裡抓出：
- **目標**（必要）：例如「00922」「台積電」「光通訊」
- **鏡頭**（可選）：若使用者有指定就用指定的；若沒指定，預設用「個股分析」一個鏡頭。使用者可同時要多個（例如「個股分析＋芒格＋塔勒布」）。

若目標不明確，先問清楚再繼續。

### 2. 依序產生每個鏡頭的分析
逐一處理選定的鏡頭（一次一個，避免同時大量請求）：

- **資料型鏡頭**：先用網路搜尋抓最新行情/財報/新聞，再分析。每段控制在約 200 字，結論先行，附一個主要風險與一個反方觀點，用繁體中文、乾淨純文字。
- **顧問型鏡頭**：呼叫對應的已安裝 nuwa skill（例如「用芒格的視角分析 00922」），讓那個 skill 用蒸餾框架回答。把它的輸出原樣收進來。

### 3. 寫一句「本次重點」
綜合這次所有鏡頭，用**一句話**（40 字內）總結這次分析的核心結論。放在這次紀錄的最上面。

### 4. 追加寫入 Markdown 檔
- 檔名：`analyses/{目標}.md`（例如 `analyses/00922.md`）。把目標裡不適合當檔名的字元換成底線。
- 若 `analyses/` 資料夾不存在，先建立。
- 若該檔**不存在**，先寫檔頭（見範本），再寫本次紀錄。
- 若該檔**已存在**，把本次紀錄**追加到檔尾**（不要覆蓋舊紀錄）——這樣同一支股票的歷史會一路累積，方便日後對照「上次怎麼看、這次怎麼看」。

### 5. 回報
告訴使用者：存到哪個檔、這是第幾次分析、本次重點那一句。

## Markdown 檔範本

新檔第一次寫入時，先放檔頭：

```markdown
# {目標} 分析紀錄

> 由「財務分析師」skill 自動產生。僅供分析參考，不構成投資建議。
> 顧問視角為 nuwa 蒸餾框架，非本人真實想法。

---
```

之後每次分析追加一段（注意最前面留一個空行分隔）：

```markdown

## {YYYY-MM-DD HH:mm}

**本次重點：** {一句話總結}

**使用鏡頭：** {列出這次用的鏡頭，例如 個股分析、芒格}

### 個股分析
{內文}

### 芒格
{munger-skill 的輸出}

...（其餘鏡頭依序）

---
```

## 邊界與注意

- **只做分析與寫檔，不執行任何交易操作。**
- 顧問視角必須來自已安裝的 nuwa skill；沒裝就說沒裝，不要假裝。
- 行情/財報數字一定要先查最新再寫，並在內文註明資料日期。
- 資產配置一律用相對權重表達，不詢問使用者實際金額。
- 這個 skill 負責「真正寫檔」；視覺化互動仍可用「蒸餾你的分析師」artifact 儀表板，兩者共用同一套分析邏輯，但儲存位置不同（artifact 存瀏覽器沙盒、此 skill 存本機資料夾）。
